本文件只保留最近任务与当前回归要点，避免历史条目干扰实现。

历史完整记录已归档到：

- `docs/records/archive/tasks_pre_v0.6.3.md`

当前生效规则以 `docs/rules/` 为准。

## 2026-03-24

### 规则基线第三轮收口（已完成：v0.6.3 文档实现优化）
- 目标：把已确认的实现风险点写死到规则，减少编码期解释空间。
- 范围：命中单位、伤害边界顺序、换人与绝对先手关系、超时比较精度、双投降结果、`invalid_battle` 结果、睡眠计时、field 冲突、来源速度快照、多来源击杀归属、30 秒选指令限时、链深配置化、记录归档拆分。
- 验收标准：开发者只读 `docs/rules/` 即可拿到唯一实现口径，`docs/records` 不再混杂大段历史规则文本。

#### 已完成内容
- 命中率口径统一为百分比 `0~100`，并写死入公式换算步骤。
- 伤害边界处理顺序固定，避免“最小伤害”和“负值保护”冲突。
- 明确手动换人低于绝对先手技能/奥义，高于普通行动。
- 超时比较增加整数交叉相乘规则，避免浮点误差。
- 增加同窗双方投降处理：`no_winner`。
- `invalid_battle` 增加固定结果语义：`no_winner` + `end_reason`。
- 睡眠改为行动窗口扣减，扣到 0 当回合苏醒并可行动。
- 同组 field 冲突改为官方思路：后结算者保留，并写死冲突裁决键。
- `source_speed` 补齐为“离场瞬间最后有效速度快照”。
- 多来源同窗致死增加 `killer` 仲裁规则。
- 增加每回合 `30s` 选指令限时与超时自动默认动作规则。
- 链深限制改为配置项 `max_chain_depth`（默认 16）。
- `docs/records` 完成归档拆分，历史全文移到 `docs/records/archive/`。

#### 最小可玩性检查清单（文档基线）
- 可启动：按文档即可实现完整一回合，不需要口头补规则。
- 可操作：超时自动动作、双投降、`invalid_battle` 终止都能走通完整流程。
- 无致命错误：排序、伤害边界、睡眠扣减、击杀归属不会因实现细节分叉。

#### 回归检查要点
- `team_hp_ratio` 是否使用交叉相乘比较而非浮点比较。
- `absolute_order > 0` 的绝对先手是否一定先于手动换人。
- 选指令超时是否会触发 `timeout_auto_action` 且日志可回放。
- 睡眠是否按行动窗口扣减，扣到 0 当回合可行动。
- 同组 field 同窗冲突是否由后结算者保留。
- `invalid_battle` 与同窗双投降是否都记为 `no_winner`。
- 链深限制是否从配置读取而不是硬编码。
