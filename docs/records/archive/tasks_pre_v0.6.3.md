### 规则基线第二轮收口（已完成：v0.6.2 基线补丁）
- 目标：把本轮你已拍板的规则正式落盘，消除剩余的实现歧义，避免开发开始后再补口头规则。

#### 最小可玩性检查清单（文档基线）
- 可启动：实现者能明确区分里程碑 A 和里程碑 B，不会第一步就把所有复杂系统绑死。
- 可操作：标准模式的公开信息、首发、回合推进、双倒补位、超时判定都有唯一口径。
- 无致命错误：不存在“是否公开”“谁先入场”“超时怎么算”“报错后怎么办”这类关键实现分叉。

#### 回归检查要点
- 里程碑 A 是否只要求最小闭环，而不是把状态/field/完整效果系统一股脑塞进第一步。
- 标准模式是否战前就完整公开队伍战斗信息。
- 同种重复单位是否带有稳定公开编号。
- 双倒补位是否先同时进入 active，再统一跑 `on_enter` 队列。
- 超时比较是否按公式而不是按口头描述实现。
- `has_started_action` 是否已经完全退场，只保留 `has_acted`。
- fail-fast 命中后是否整场战斗立即终止并保留日志。

### 战斗规则 v0.6 收口与赛事化重构（已完成：v0.6 基线收口稿）
- 目标：把当前规则重新收口成单一方向，消除“剧情战背包道具”和“官方思路项目化改写”混搭造成的口径摇摆。
- 范围：战斗定位、战前公开信息、道具系统、回合上限、闪避边界、技能合法性、非行动链路、同触发点排序、MP 时序、文档一致性同步。
- 验收标准：`docs/rules/` 与 `docs/records/decisions.md` 对上述问题只有一套口径，不再需要靠聊天补规则。

#### 已完成内容
- 将标准战斗定位收口为“参考官方思路的项目化改写”。
- 将战前信息改为“公开双方队伍名单，不公开技能、持有物与详细数值”。

- 将 `max_turn` 从 100 改为 40。

#### 最小可玩性检查清单（文档基线）
- 可启动：开发者只读 `docs/rules/` 就能理解当前标准模式不再包含主动道具。
- 可操作：至少能覆盖“公开队伍名单 -> 选首发 -> 选技/换人 -> 结算 -> 击倒补位 -> 回合末 -> 超时判定”的完整路径。
- 无致命错误：非行动系统链、触发排序、技能合法性、MP 时序没有口头补充依赖。

#### 回归检查要点
- 战前是否公开敌方 3 人名单，但仍隐藏技能与持有物。
- 标准模式中是否彻底没有主动道具指令入口。
- 回合上限是否为 40，且超时比较顺序未变。
- 常规闪避是否不会直接堆到 `1.0`，只有特殊效果可以。
- 满血治疗、空放净化是否允许进入执行阶段后失败，而不是在选择阶段被拦截。
- `turn_start` 是否先回 MP 再跑 `turn_start` 触发；`turn_end` 是否先跑触发再回 MP。
- `battle_init / turn_start / turn_end / timeout / system_replace` 是否都能创建独立链路并写入日志。
- 同触发点排序是否按 `effect.priority -> source_speed -> source_kind_order -> source_instance_id -> random` 执行。

### 战斗系统模块化拆解与定稿（已完成：v0.2 基线冻结，历史记录）
- 目标：完成战斗系统模块化拆解与规则定稿，先冻结可实现、可验证、可回归的基线。

- 验收标准：所有模块有明确规则，且关键边界条件（资源、双倒下、持续时间、递归触发）有统一定义。

#### 已定稿模块
- 回合流程与胜负判定（含同结算窗口双倒下平局）
- 队伍与上场规则
- 指令系统（含 MP 不足不可选、默认动作）

- 伤害公式（升级为官方骨架 + 原型简化）
- 伤害边界规则（最小伤害、免疫、负值保护）
- 状态与能力变化（持续时间扣减时点统一）

- 全局效果（field 最小版本）

- 效果系统对接规范（含换人与入场触发顺序）

#### 最小可玩性检查清单（基线）
- 可启动：可进入战斗并完成回合推进。
- 可操作：至少完成一次“选技 -> 结算 -> 回合末 -> 下一回合”的完整循环。

- MP 回复时机（turn_start/turn_end）与上限截断正确。

- 效果持续时间“当回合不扣减、下回合末扣减”一致。
- `on_resource_change` 链路不会出现无限递归。
- 换人与入场触发顺序固定且可复现。

#### 最小可玩性检查清单（文档基线）
- 可启动：开发者仅阅读 `docs/rules/` 即可开始实现战斗主流程，不需要额外口头补规则。
- 可操作：至少能覆盖“盲选首发 -> 选技/换人/主动道具 -> 结算 -> 击倒补位 -> 回合末 -> 胜负/超时”的完整路径。
- 无致命错误：离场、双倒、状态阻断、默认动作、field 覆盖、日志归属均有唯一口径。

#### 回归检查要点（v0.5 文档体系）
- 当前生效规则是否都迁移到 `docs/rules/`，而不是继续写在 `docs/records/`。
- 队伍构筑是否为“同种不限 + 盲选 + 无 Team Preview”。
- AI 是否只允许读取公开信息。
- 行动队列是否在选择阶段后锁定，回合内不重排。
- 被睡眠/麻痹/畏缩拦下时，是否记为“行动机会过去但没行动成”。
- 倒下离场是否不会触发 `on_switch`。
- 离场重置表是否覆盖阶段、护盾、易失状态、锁定标记。
- 睡眠是否改为由技能/效果定义持续时间，而不是写死全局回合数。
- 同窗双倒是否先统一标记倒下，再统一判断补位/平局。
- 主动道具是否必须在选择阶段验证“确实会改变战场状态”。
- field 是否按“后结算覆盖前结算”工作。
- 日志是否包含 `action_id` 与 `leave_reason`。
