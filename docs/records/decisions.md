# 决策记录

本文件记录关键设计决策（概念与原型期）。

> 更新说明：自 v0.5 起，当前生效规则请查看 `docs/rules/`。本文件只记录“为什么这样定”，不再承载现行细则。

## 2026-03-24

> 说明：同日后续编号决策可覆盖前序决策的细节口径；若有冲突，以编号更大的决策为准。

### 1. 战斗系统范围收敛
- 仅做战斗系统，不包含探索与捕获等内容。

### 2. 单打与队伍结构
- 单打 1v1，在场各 1 名单位。
- 每队 3 名单位，战前选择首发。
- 倒下后强制换下一名单位上场。
- 移除投降指令。

### 3. 指令系统
- 指令为：技能、换人、道具、必杀/奥义。
- 换人与道具均占用行动。
- 必杀/奥义保留“先手型/后手型”与发动条件。

### 4. 行动顺序与优先级
- 判定链固定：绝对层级 → 优先级 → 速度 → 随机。
- 基准优先级为 0；换人、普通技能、道具默认 0。
- 其他技能优先级范围为 -2 到 +2。

### 5. 数值体系
- 采用标准六维：HP、攻击、防御、特攻、特防、速度。
- 保留独立闪避率设计。

### 6. 命中规则（边界保护）
- 技能命中率默认 100%。
- 命中判定公式为：`clamp(技能命中率 × (1 - min(目标闪避率, 1.0)), 0, 1)`。
- 维持当前独立闪避率方案，不切换到官方命中阶段制。

### 7. 资源系统（MP）闭环
- MP 为角色独立资源池。
- 角色资源字段明确为：`max_mp`、`mp`、`init_mp`、`regen_per_turn`、`regen_timing`。
- `regen_timing` 支持 `turn_start` / `turn_end`，默认 `turn_start`。
- MP 回复后不得超过 `max_mp`。

### 8. MP 不足时的指令规则
- MP 不足的技能不可选。
- 若角色本回合无可执行主动技能且无其他可执行指令，自动执行默认动作（挣扎，0 MP 消耗）。

### 9. 伤害系统升级为官方骨架（原型简化）
- 基础伤害采用官方骨架公式（Level 参与）。
- 原型期固定 `Lv=50`。
- 随机系数采用 16 档 `0.85~1.00`。
- 属性克制未启用阶段按 `1.0` 处理，预留后续接入点。
- 保留全局系数 `K` 作为整体调参入口。

### 10. 伤害边界规则
- 统一向下取整。
- 防御参与除法时下限为 1。
- 非免疫且命中时伤害至少为 1。
- 免疫时伤害为 0。
- 任何后处理导致的负伤害按 0 结算。

### 11. 真实伤害规则
- 真实伤害无视防御与特防。
- 真实伤害不可被护盾抵挡。

### 12. 双倒下判定
- 同一结算窗口内若双方均无可上场单位，判定平局。

### 13. 状态与能力阶段
- 状态系统采用统一效果实例框架。
- 能力阶段维持简化版（-2 到 +2）。

### 14. 持续时间结算统一
- `turns` 模式的效果持续时间在回合末结算后扣减。
- `turns` 模式下新施加效果当回合不扣减，从下一次回合末开始扣减。
- 扣减后 `remaining<=0` 立即移除。

### 15. 事件防循环
- 每次行动建立 `event_chain_id`。
- 同一链路去重规则后续已在第 29 条细化（以第 29 条为准）。
- 连锁触发深度超过上限（默认 16）立即报错并中断当前链。
- `on_resource_change` 链路同样受上述限制。

### 16. 被动系统
- 每角色 1 个被动，不占技能槽。
- 被动允许造成伤害，触发点支持扩展。

### 17. 道具系统约束
- 每场战斗最多携带 3 个道具。
- 不可携带同名道具。
- 每回合每方最多主动使用 1 个道具，且占用行动。
- 支持常驻（战前携带即生效）效果。

### 18. 全局效果（field）
- 引入 field 作用域作为全局规则统一入口。
- 允许多种全局效果并存，互斥由 `exclusive_group` 控制。

### 19. 换人与入场触发顺序固定
- 顺序固定为：`on_switch -> on_exit -> 入场 -> on_enter -> 入场引发效果结算`。
- 同触发点内结算顺序为：`priority -> 速度 -> 随机`。

### 20. 奥义资源规则暂缓
- 奥义资源的获得、上限、消耗、清空规则在角色技能设计阶段单独定稿。

### 21. 可见性、AI 与日志
- 敌方 MP 可见。
- 被动触发后显示。
- 敌方 AI 采用权重策略。
- 日志需详细记录关键结算节点。

### 22. 战斗 UI
- UI 布局参考宝可梦战斗界面。
- 控制台输出全量日志，UI 显示摘要。

### 23. 非法行动拦截与重选时机
- 非法行动必须在“选择指令阶段”被拦截，并提示玩家重新选择合法行动。
- 进入执行阶段后不允许重选；若行动因战场变化在执行时失效，则该行动失败并消耗本次行动。

### 24. 持续时间模式统一
- 增加 `duration_mode`：`turns / triggers / permanent`。
- `turns` 在回合末扣减；`triggers` 在对应触发成功后扣减；`permanent` 不自动扣减。

### 25. 奥义入口状态
- 奥义资源规则将在“角色技能设计”阶段定稿。
- 在定稿前，奥义指令入口默认关闭，仅保留接口与排序扩展位。

### 26. 回合上限与超时判定
- 设定 `max_turn = 100`，防止无限对局。
- 达到回合上限后按“存活单位数 -> 全队剩余 HP 总和 -> 平局”判定胜负。

### 27. 真实伤害与结算优先级
- 真实伤害无视防御与特防。
- 真实伤害不可被护盾抵挡。
- 统一伤害结算顺序：命中 -> 免疫 -> 基础伤害 -> 修正 -> 护盾 -> 边界保护与扣血。

### 28. 常驻效果叠加规则
- 维持“不可携带同名道具”约束。
- 常驻效果允许叠加，但仅 `stacking=stack` 的效果参与叠层。
- 按 `effect_id` 计层，`max_stacks` 默认 3（可单独配置）；超过上限不再加层，可按定义刷新持续时间。

### 29. 事件防循环去重键优化
- 单链路去重从“同 Trigger 只触发一次”升级为：`effect_instance_id + trigger + event_step_id`。
- `event_step_id` 用于区分多段命中等合法重复触发场景（如 `hit_index`）。

### 30. 日志可复现性增强
- 日志必须记录 `battle_seed`、`turn_index`、`hit_roll`、`crit_roll`、`damage_roll`（16 档）等关键随机信息。
- 继续保留行动选择、伤害值、状态变化、资源变化、击倒/换人事件的完整记录要求。

### 31. 单一规则源固定
- 历史口径：`battle_system_rules.md` 曾作为唯一执行规范来源；自 v0.5 起已迁移为 `docs/rules/` 文档体系。
- `decisions.md` 仅记录决策背景与原因。
- `tasks.md` 仅记录任务过程与验收结果。
- 历史口径：当时定义“三者冲突时，以 `battle_system_rules.md` 为准”；自 v0.5 起改为“当前生效规则以 `docs/rules/` 为准”。

### 32. 行动排序层级收口
- 行动排序统一为：`absolute_order -> action_priority -> skill_priority -> speed -> random`。
- `absolute_order` 仅保留给绝对先手型 / 绝对后手型奥义接口。
- 手动换人与主动道具默认 `action_priority = +1`。
- 普通技能默认 `action_priority = 0`，其自身先后手差异由 `skill_priority` 表达。

### 33. 击倒窗口前移
- 每个行动结算后立即检查击倒。
- 若有单位倒下，按 `on_faint -> 强制补位 -> on_enter -> 入场衍生效果` 立即处理。
- 强制补位不参与本回合行动排序，属于系统动作。

### 34. 单体目标锁定口径
- 单体敌方技能默认锁定敌方在场位置，不锁定具体旧单位。
- 若执行时该位置已有新单位，则命中新单位。
- 若执行时该位置为空，则行动失效并消耗本次行动。

### 35. 道具系统拆分为主动 / 被动子系统
- 保留道具系统，不改为纯持有物方案。
- 主动道具与被动道具为两个子系统，不共享库存与装备位。
- 主动道具：每方每战最多携带 3 个，不可重复，占用行动，默认 `action_priority = +1`，每方每回合最多主动使用 1 次。
- 被动道具：战前装备，每单位最多 1 个，同队不可重复，不占行动。
- 主动与被动之间不做跨类型重名限制。

### 36. 状态最小包冻结
- 主要状态冻结为：中毒、灼伤、麻痹、睡眠。
- 易失状态冻结为：畏缩。
- 历史口径：当时写为“同一单位同时最多存在 1 个主要状态”；自 v0.5 起改为“通过 `exclusive_group` 控制互斥，当前内置主要状态默认同组互斥，但框架不再硬编码死规则”。
- 状态默认不因换人自动清除；离场期间不参与回合节点结算，回到场上后继续生效。

### 37. 护盾、治疗与复活边界
- 护盾为独立吸收池，普通伤害先扣护盾，溢出再扣 HP。
- 真实伤害跳过护盾。
- 治疗不得超过 `max_hp`。
- 倒下单位默认不可被治疗。
- 原型阶段关闭复活效果与复活类道具。

### 38. 闪避上限与命中标签
- 保留独立闪避率方案，且有效闪避率上限为 `1.0`。
- 增加 `cannot_miss`（必中）与 `ignore_evasion`（无视闪避）标签。
- 允许完全闪避，具体强度由内容设计阶段控制。

### 39. 暴击语义收口
- 暴击仅作为伤害倍率。
- 暴击不额外穿透防御、护盾、能力阶段或其他减伤逻辑。

### 40. 回合上限与随机顺序收口
- 达到 `max_turn = 100` 后，按“存活单位数 -> 全队平均剩余 HP 百分比 -> 全队剩余 HP 总和 -> 平局”判定。
- 随机抽样顺序固定为：`speed_tie_roll -> hit_roll -> crit_roll -> damage_roll -> effect_roll`。
- 多段命中需记录 `hit_index`，并为每段分别记录对应随机抽样值。

### 41. 规则文档迁移为“总则 + 模块权威文档”
- 不再由 `docs/records/battle_system_rules.md` 单文件承载当前生效规则。
- 当前生效规则迁移到 `docs/rules/` 目录。
- `docs/rules/00_rule_baseline.md` 负责总则；`docs/rules/01~06` 负责模块细则。

### 42. 构筑与开局口径补齐
- 当前原型仍为单打 1v1、每队 3 名单位。
- 当前不限制同种单位。
- 当前战前为盲选，不做 Team Preview，不做 Open Team List。

### 43. 行动队列锁定
- 双方合法指令确认后，一次性生成本回合行动队列。
- 回合内即使速度、地形、状态发生变化，也不重排本回合已锁定队列。

### 44. “行动机会已过去”与“行动成功开始执行”拆开
- 新增 `action_window_passed` 用于判断本回合该单位的行动时机是否已经轮过。
- 新增 `has_acted` 用于判断本次行动是否真的成功开始执行。
- 被睡眠、麻痹、畏缩拦下时，记为“行动机会已过去，但没有行动成”。

### 45. 行动失败分类写死
- 区分：非法指令、自动替代默认动作、未开始就取消、开始前被状态拦下、开始后失效、命中失败、成功结算。
- 技能/奥义到达自己的行动窗口时先锁定并扣 MP，再判定睡眠/麻痹/畏缩等前置阻断；若被拦下，资源不返还。
- 换人和主动道具默认不受睡眠、麻痹、畏缩阻断，因为它们按 trainer 行为处理。

### 46. 离场原因与触发顺序正式区分
- `leave_reason` 固定为 `manual_switch / forced_replace / faint`。
- 倒下离场不触发 `on_switch`。
- 强制换下（未倒下）会触发 `on_switch` 和 `on_exit`。

### 47. 离场重置表收口
- 长期状态默认不会因普通换人自动清除。
- 易失状态、能力阶段、个人护盾、锁定目标、蓄力标记等离场即清。
- team / field 效果不因单个单位离场而消失。

### 48. 状态系统改为“分组互斥 + 数据驱动时长”
- 不再把“永远只能有一个主要状态”写死成框架死规则。
- 当前内置主要状态默认同属 `major_status` 互斥组，因此当前仍互斥。
- 睡眠时长不写死全局值，由施加它的技能/效果定义。

### 49. 同窗双倒下与窗口定义补齐
- 固定四类窗口：单次效果窗口、行动后窗口、回合末窗口、回合上限窗口。
- 同一窗口内若双方都无可上场单位，则判平局。
- 若同窗双方同时倒下，先统一标记倒下并处理离场，再统一判断是否补位或平局。

### 50. 主动道具合法性写死
- 主动道具必须在选择阶段满足“确实会改变战场状态”，否则非法。
- 主动道具默认不受睡眠、麻痹、畏缩阻断。
- 行动者若在道具行动轮到前倒下，道具不消耗。

### 51. field 冲突口径写死
- 同一 `exclusive_group` 只允许 1 个 field 生效实例。
- 同组不同 field 采用“后结算覆盖前结算”的口径。
- 这条参考官方天气/场地“新效果覆盖旧效果”的思路。

### 52. AI 只允许读取公开信息
- AI 只能读取“公开信息”与自己一侧的完整私有信息。
- AI 禁止直接读取敌方隐藏后备、未揭示技能、未揭示被动道具和内部随机。

### 53. 事件归属与日志字段补齐
- 统一 `source / owner / target / killer / creator` 的定义。
- DOT、field、反伤、自伤都必须有明确归属规则。
- 新增 `action_id`、`leave_reason` 等日志字段，保证可回放与可追责。

### 54. 伤害修正层拆分
- 不再用“其他修正”一个大口袋兜底。
- 伤害层固定拆为：命中、免疫、基础伤害、前置修正、暴击、随机、STAB、属性、最终修正、护盾、边界。

### 55. 默认动作采用“参考官方思路的项目化改写”
- 当所有主动技能都非法，且无合法换人/道具/奥义时，自动执行默认动作。
- 默认动作参考官方 Struggle 的“资源耗尽也要能出手”思路，但数值按本项目规则单独定义。

### 56. 战斗体验定位改为“接近官方赛事式公开博弈”
- 当前标准战斗的体验目标改为更接近官方赛事战，而不是剧情战的背包道具对战。
- 在此基础上继续保留本项目自己的主动分叉，例如 1v1、每队 3 人、共享 MP、简化队伍预览。

### 57. 战前信息改为“公开队伍名单，不公开技能与持有物”
- 不再使用完全盲选。
- 战斗开始前，双方公开 3 名单位名单与基础展示信息。
- 技能、持有物、详细数值仍保持隐藏，不采用完整 Open Team List。

### 58. 标准模式移除主动道具，只保留被动持有物
- 当前标准战斗基线不启用主动道具指令。
- 道具系统只保留被动持有物：每单位最多 1 个，同队不可重复，战前配置，战斗中不占行动。
- 若未来要做剧情/PVE 的主动道具模式，必须作为独立模式另立规则，不默认并回当前标准基线。

### 59. 回合上限改为 40
- 当前标准战斗的 `max_turn` 从 100 改为 40。
- 达到上限后仍按“存活单位数 -> 全队平均剩余 HP 百分比 -> 全队剩余 HP 总和 -> 平局”判定。

### 60. 常规闪避不允许直接满闪，只有特殊效果能到 1.0
- 常规来源累计后的有效闪避率上限固定为 `0.5`。
- 只有技能或效果显式声明 `allow_full_evasion` 或等效特殊标记时，才允许把有效闪避率提升到 `1.0`。

### 61. 持续效果时长继续保持技能定义，不设全局统一值
- 睡眠与其他持续效果的时长，继续由技能或效果实例自己定义。
- 规则层不再额外补全局统一时长上限。

### 62. 技能选择阶段只拦截硬非法，其余允许执行失败
- MP 不足、目标位置不存在、替补不存在、奥义入口关闭、标准模式未启用的指令类型等，属于硬非法，不能选。
- 满血治疗、空放净化、对当前无可驱散对象使用驱散等，不属于硬非法，允许玩家自己选，进入执行阶段后按失败分支处理。

### 63. 若未来恢复主动道具模式，每个主动道具必须声明消耗时机
- 当前标准模式下，这条不生效，因为主动道具已移出基线。
- 若未来重开主动道具模式，每个主动道具都必须单独声明 `consume_timing`，不能再靠全局一句话兜底。

### 64. 非行动系统段也必须建立链路 ID
- `battle_init / turn_start / turn_end / timeout / system_replace` 等非行动系统段，同样必须建立 `event_chain_id`。
- 完整日志中新增 `chain_origin`，用于区分链路来源。

### 65. 同触发点统一排序补齐到完整来源集合
- 同触发点排序统一为：`effect.priority -> source_speed -> source_kind_order -> source_instance_id -> random`。
- `source_speed` 固定为速度高者先；若来源已离场则取创建快照速度；系统来源按 `0` 处理。
- `source_kind_order` 固定为：`system -> field -> active_skill -> passive_skill -> passive_item -> status_effect`。

### 66. MP 回复时序固定
- `turn_start` 的 MP 回复先于 `turn_start` 效果队列。
- `turn_end` 的 MP 回复后于 `turn_end` 效果队列。
