# 决策记录（精简版）

本文件只保留当前阶段仍需频繁引用的关键决策。

历史完整记录已归档到：

- `docs/records/archive/decisions_pre_v0.6.3.md`

当前生效规则仍以 `docs/rules/` 为准；本文件只记录“为什么这样定”。

## 2026-03-24

### 83. 超时比较精度改为整数比较
- `team_hp_ratio` 不再用浮点直接比较，统一改为交叉相乘比较，避免跨语言与平台浮点误差导致判定分叉。

### 84. 手动换人排序低于绝对先手奥义
- 手动换人保持 `action_priority = +1`，但固定低于 `absolute_order > 0` 的绝对先手技能/奥义。
- 手动换人仍高于普通技能、默认动作与非绝对先手奥义。

### 85. 增加每回合选指令 30 秒限时
- 每方每回合 `30s`，超时未提交合法指令则自动替代默认动作。
- 自动替代必须写入日志并可回放。

### 86. 睡眠计时按官方思路收口
- 睡眠改为“行动窗口计时”：在目标自己的行动窗口开始时扣减。
- 扣减到 0 的当回合立即苏醒并可继续行动。
- 换人不会自动清除睡眠，重新上场继承剩余睡眠回合。

### 87. 同组 field 冲突按官方天气/场地思路处理
- 同窗内都落地时由后结算者保留。
- 同组 field 冲突裁决时不额外比较 `effect.priority`，避免实现分叉。

### 88. 异常终止与同窗双投降统一为无胜者
- `invalid_battle` 统一结果为 `no_winner`，并写入 `end_reason = invalid_battle`。
- 同一结算窗口双方都投降时，同样按 `no_winner` 结束。

### 89. 命中单位与伤害边界顺序写死
- `accuracy` 明确为百分比口径 `0~100`，参与命中公式前统一换算为小数。
- 伤害边界按固定顺序处理：免疫/未命中 -> 负值保护 -> 最小伤害 -> 最终扣血，避免歧义。

### 90. 来源速度快照与多来源击杀归属补齐
- 来源已离场时，`source_speed` 取离场瞬间的最后有效速度快照。
- 同窗多来源致死时，按“最后致死事件”优先；并列再按统一排序键仲裁 `killer`。

### 91. 信息可见性增加模式档位
- 当前默认为 `prototype_full_open`。
- 额外预留 `official_open_team_list` 档位，便于后续贴近官方赛事公开口径。

### 92. 链深限制改为可配置
- 链深限制不再视为硬编码常量，统一走配置项 `max_chain_depth`。
- 原型默认值保留 16，但允许按模式调参。
