# 模块 01：战斗模式、队伍构筑与信息可见性

本文件定义“打什么、怎么开局、双方能看到什么”。

## 1. 战斗模式

|项|规则|
|---|---|
|战斗形式|单打 1v1|
|队伍规模|每队 3 名单位|
|等级口径|战斗内按 `Lv = 50` 计算|
|首发数量|每方 1 名|
|投降|提供；触发后立即判该方负、对方胜|
|每回合选指令限时|每方 `30s`|
|回合上限|`max_turn = 40`|

## 2. 队伍构筑

### 2.1 当前原型构筑限制

|项|规则|
|---|---|
|同种单位|当前不限制，可重复|
|同名被动持有物|同一队伍内不可重复|
|每单位被动持有物|最多 1 个|
|主动道具|标准模式不启用|
|技能槽|常规技能 3 个，被动 1 个，奥义槽保留但入口关闭|

### 2.2 当前明确不采用的官方比赛限制

|官方常见限制|当前口径|
|---|---|
|同国家图鉴号不可重复|当前不启用|
|官方 Open Team List 文书字段清单|当前不逐字段照搬，但标准模式采用更彻底的“完整公开队伍战斗信息”|
|只做 Team Preview 不做完整公开|当前不采用；本项目直接在开战前公开完整队伍战斗信息|

## 3. 队伍信息公开、首发与开局

|项|规则|
|---|---|
|战前公开信息|标准模式 `prototype_full_open` 下，双方公开 3 名单位的完整战斗信息：单位身份、等级、技能列表、被动、被动持有物、基础战斗数值、公开编号|
|首发选择方式|战斗开始前，各方从自己的 3 名单位中选择 1 名首发|
|是否盲选|否。首发选择前可见敌方完整队伍信息|
|是否公开技能/持有物|是，战前即完整公开|
|同种重复标识|若同队存在重复单位，必须为每个单位分配稳定公开编号，如 `P1-A / P1-B / P1-C`，用于预览、日志与回放|
|是否可反复改首发|在战斗真正开始前允许改；进入战斗初始化后锁定|
|首发确认后|双方首发同时入场，随后处理 `on_enter` 与入场衍生效果|

## 4. 公开信息与私有信息

### 4.1 玩家可见信息

|信息|是否公开|说明|
|---|---|---|
|敌方完整队伍单|公开|单位身份、公开编号、等级、技能、被动、被动持有物、基础战斗数值战前即公开|
|敌方当前在场单位身份|公开|单位名、立绘、公开编号与基础展示信息可见|
|敌方在场单位当前 HP 百分比/精确值|公开|当前标准模式允许 UI 同时显示占比与精确值|
|敌方在场单位当前 MP|公开|当前系统明确公开|
|敌方在场单位已显现状态|公开|包括中毒、灼伤、麻痹、睡眠、畏缩等|
|已生效的 field / team 效果|公开|剩余回合、层数、来源在日志中可见|
|敌方被动持有物|公开|标准模式战前即完整公开，不再等待触发揭示|
|敌方技能列表|公开|标准模式战前即完整公开|
|敌方后备单位名单|公开|战前公开 3 名单位名单后，整场保持公开|
|随机抽样值|隐藏|只写入日志，不在常规 UI 公开|
|未来未发生的随机结果|隐藏|AI 与玩家都不能提前读取|

### 4.2 AI 信息边界

|项|规则|
|---|---|
|AI 读取范围|标准模式下可读取全部战斗公开信息 + 自己一侧运行态|
|AI 禁止读取|内部随机结果、未来未发生的抽样值、仅供调试的内部缓存字段|
|AI 允许使用|直接基于完整公开队伍信息做决策，不需要再维护“敌方隐藏招式/持有物猜测表”|

### 4.3 信息可见性模式档位

|模式 ID|用途|公开范围|
|---|---|---|
|`prototype_full_open`|当前默认标准模式，用于原型期快速验证规则|公开队伍名单、技能、被动、持有物、基础战斗数值、在场精确 HP/MP|
|`official_open_team_list`|贴近官方 Open Team List 的赛事化档位（可选）|公开队伍名单、技能、特性/被动、持有物、太晶属性/等效字段；不公开基础战斗数值与内部随机|

补充规则：

1. 当前文档其余章节默认按 `prototype_full_open` 执行。
2. 若切换到 `official_open_team_list`，必须在对局创建参数与日志头显式写明模式 ID。

## 5. 战斗开始时的初始化顺序

|步骤|规则|
|---|---|
|1|加载双方队伍与被动持有物配置|
|2|公开双方完整队伍战斗信息与稳定公开编号|
|3|各方选择首发|
|4|首发同时入场|
|5|创建战斗日志头：`battle_seed`、队伍快照、首发快照|
|6|处理双方 `on_enter` 与入场触发的被动 / 持有物 / field|
|7|若初始化结算导致单位倒下，立刻进入击倒窗口与强制补位|
|8|进入第 1 回合开始阶段|

## 6. 胜负与结束条件（格式层）

|项|规则|
|---|---|
|通常胜利|对方无可上场单位|
|通常失败|己方无可上场单位|
|同窗双空|同一结算窗口双方都无可上场单位，则判平局|
|同窗双投降|同一结算窗口双方都触发投降，则按 `no_winner` 结束|
|回合上限结束|第 40 回合完整结算后执行超时比较|

补充规则：

1. 投降属于即时结束战斗的认负行为，不进入行动队列，不参与优先级比较。
2. 投降后仍需写入完整日志（触发方、触发回合、结算结果）。
3. 若同一结算窗口内双方都触发投降，则本场按 `no_winner` 结束（平局语义）。

## 7. 平衡约束声明

当前原型阶段不对“构筑平衡”做承诺。优先目标是：

1. 规则口径唯一。
2. 行为可预测。
3. 日志可回放。
4. 首个可玩闭环稳定。
