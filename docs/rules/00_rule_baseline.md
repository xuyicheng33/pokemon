# 战斗系统规则总则（v0.7.0）

本文件定义当前原型期战斗规则的唯一基线。目标不是一次把所有机制做满，而是先冻结一套简单、可玩、可实现、可回放的闭环。

## 1. 范围与目标

|项|规则|
|---|---|
|项目阶段|概念与原型期|
|当前范围|仅战斗系统，不包含探索、捕获、养成|
|当前目标|先做出 1v1 回合制战斗最小闭环，保证规则唯一、实现清楚、玩家能快速看懂|
|设计优先级|规则唯一 > 新手易懂 > 可玩闭环 > 扩展能力 > 数值平衡|
|实现原则|开发阶段默认 fail-fast；非法输入直接拦截；发现文档冲突先修文档再开发|

## 2. 权威文档边界

|项|规则|
|---|---|
|总则职责|定义全局不变量、统一术语、主动分叉与模块边界|
|模块职责|给出可直接编码的细则|
|记录文档职责|只记录“为什么这样定”“这次任务做了什么”，不承载当前生效规则|
|冲突处理|模块细则优先于总则摘要；两个模块互相打架时，视为文档缺陷，先修文档|

## 3. 当前主动分叉

这些是当前版本主动写死的设计，不是“以后再看”的临时说法：

|分叉点|当前规则|
|---|---|
|资源系统|使用单位独立 `MP`，不使用官方 `PP`|
|战斗规模|单打 1v1，每队 3 名单位|
|信息公开|当前只保留 `prototype_full_open`，战前公开完整队伍战斗信息|
|道具系统|标准战斗只保留被动持有物，不存在主动使用道具指令|
|状态系统|当前标准基线不内置通用状态包；若以后某个技能需要持续效果，按技能定义单独写清|
|命中体系|只看技能自身 `accuracy`，当前没有闪避率、命中阶段、闪避阶段|
|伤害体系|只保留物理 / 特殊两类普通伤害；当前没有暴击、真伤、护盾、属性克制、同属性加成|
|场地系统|保留 `field`；当前同一时刻全场只允许 1 个 field 生效，新 field 直接替换旧 field|
|优先级体系|统一只有一个 `priority` 数轴；数值越大越先；行动与效果都共用这套语义|

## 4. 统一术语

|术语|定义|
|---|---|
|单位 `unit`|可参战角色个体|
|方 `side`|玩家方或敌方|
|在场位 `active_slot`|当前 1v1 场上的唯一位置|
|后备位 `bench`|未在场但仍可上场的单位列表|
|行动 `action`|本回合一次合法进入行动队列的指令|
|效果 `effect`|由技能、奥义、被动、持有物、field 触发的结算单元|
|公开信息 `public info`|玩家和 AI 都允许读取的信息|
|私有信息 `private info`|随机值、未来抽样结果、调试字段等运行态内部信息|
|同一结算窗口|同一次行动后的击倒检查、同一次回合末结算、或同一次系统效果批次|

## 5. 全局实现原则

|项|规则|
|---|---|
|唯一可见性模式|当前只支持 `prototype_full_open`|
|队列锁定|双方都提交合法指令后，行动队列一次性锁定；本回合中途不重排|
|非法处理|非法指令必须在选择阶段拦截，不得带着非法状态进入执行阶段|
|系统动作|强制补位不进入行动队列，在击倒窗口内立即处理|
|投降|即时结束，不进入行动队列，不参与 `priority` 比较|
|统一优先级语义|所有 `priority` 都是“数值越大越先”，默认值都是 `0`|
|行动排序|`priority -> speed -> random`|
|效果排序|`priority -> source_order_speed_snapshot -> source_kind_order -> source_instance_id -> random`|
|RNG 契约|所有随机判定必须使用同一确定性 RNG，按消费顺序写入完整日志|
|日志目标|完整日志必须足够回放整场战斗，公开日志只负责给玩家看懂发生了什么|
|玩家上手原则|规则层可以 full open，但 UI 默认展示必须分层，避免把全部细节一股脑砸给玩家|

## 6. 当前最小可玩闭环

|模块|必须落地|
|---|---|
|主流程|战斗开始、首发、选指令、技能、换人、奥义、击倒补位、结束判定|
|数值|HP、攻击、防御、特攻、特防、速度、MP|
|命中|只看技能 `accuracy`|
|伤害|参考官方骨架的简化版物理 / 特殊伤害|
|道具|被动持有物|
|field|单全局 field，按技能描述生效|
|日志|至少记录种子、回合、行动、命中、伤害、HP/MP 变化、field 变化、离场、胜负|

## 7. 当前明确不做

|内容|口径|
|---|---|
|通用状态包|当前不做中毒、灼伤、麻痹、睡眠、畏缩等内置状态|
|主动道具|当前不做|
|闪避率|当前不做|
|暴击|当前不做|
|真实伤害|当前不做|
|护盾|当前不做|
|属性克制 / STAB|等属性系统接入后再单独补规则|
|多档信息可见性|当前不做 `official_open_team_list` 等其他模式|

## 8. 文档变更纪律

|项|规则|
|---|---|
|新增规则|必须写入 `docs/rules/` 对应模块|
|跨模块改动|先改总则，再改模块，再补 `docs/records/decisions.md`|
|历史文件|旧记录可以保留，但不得再混入现行规则正文|
