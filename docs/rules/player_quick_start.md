# 玩家一页看懂：当前战斗玩法（v0.7.0）

本文件是玩家速览，不是权威规则正文。  
实现与评审一律以 `docs/rules/00~06` 为准。

## 1. 一局怎么玩

|项|当前玩法|
|---|---|
|对战形式|1v1 回合制|
|队伍规模|每队 3 名单位|
|胜利条件|对方没有可上场单位|
|回合上限|40 回合，超时后按统一规则比较|
|投降|可以，点了就立刻判负|

## 2. 每回合流程（玩家视角）

1. 回合开始：当前出战单位先回复 MP。
2. 选指令：你在 30 秒内选技能、换人或奥义。
3. 排行动顺序：先比 `priority`，再比速度，最后同速随机。
4. 执行动作：按顺序结算命中、伤害、效果。
5. 击倒检查：有人倒下就立刻补位。
6. 回合结束：结算回合末效果与 field 剩余回合。

## 3. 行动先后（统一 `priority`）

|行动|`priority`|
|---|---|
|绝对先手奥义|`+5`|
|换人|`+4`|
|普通技能|`-2 ~ +2`（默认 `0`）|
|资源/超时默认动作|`0`|
|绝对后手奥义|`-5`|

统一规则只有一句：`priority` 数值越大，越先行动。

## 4. 命中和伤害（简化宝可梦骨架）

|项|当前规则|
|---|---|
|命中|只看技能自己的 `accuracy`|
|闪避|当前没有闪避率机制|
|伤害类型|分物理和特殊|
|物理攻防|攻击方攻击 vs 防守方防御|
|特殊攻防|攻击方特攻 vs 防守方特防|

伤害核心公式：

```text
base_damage = floor(floor(((2 * Lv / 5 + 2) * Power * A / max(1, D))) / 50) + 2
final_damage = max(1, floor(base_damage * final_mod))
```

`Power` 就是技能威力。当前战斗固定 `Lv = 50`。

## 5. 持有物和 field

|系统|当前规则|
|---|---|
|持有物|只有被动持有物，战前装备，不是行动指令|
|field|全场同一时刻最多 1 个，新 field 会覆盖旧 field|
|field 效果来源|必须写在技能或效果描述里，按描述生效|

## 6. 当前还没有的机制

|机制|状态|
|---|---|
|通用状态包（中毒/睡眠等）|当前不做|
|暴击|当前不做|
|真实伤害|当前不做|
|护盾|当前不做|
|属性克制 / STAB|下一阶段属性系统接入后再补|
|主动道具指令|当前不做|

## 7. 新玩家开局只看这 4 个信息就够

1. 你和对手当前出战单位的 HP/MP。
2. 你这回合可选技能的人话描述。
3. 当前 field 是什么，还剩几回合。
4. 你的奥义是绝对先手还是绝对后手。
