# 模块 04：状态、换人、离场与生命周期

本文件定义“状态怎么挂、离场清什么、倒下和换人分别触发什么”。

## 1. 状态系统总原则

|项|规则|
|---|---|
|实现方式|状态统一走效果实例框架，不做散装特判|
|互斥方式|不写死“永远只能有一个主要状态”，而是通过 `exclusive_group` 控制|
|当前内置主要状态|中毒、灼伤、麻痹、睡眠|
|当前内置易失状态|畏缩|
|当前内置主要状态分组|以上四种默认都属于 `exclusive_group = major_status`，因此当前互斥|
|框架扩展性|未来允许新增不在 `major_status` 组里的其他长期状态|

补充规则：

- 当前内置状态尽量参考官方主系列语义。
- “当前内置主要状态互斥”是当前基线，不等于框架永久只允许一个长期状态。

## 2. 状态施加与互斥

|场景|规则|
|---|---|
|目标没有同组状态|正常施加|
|目标已有同 `exclusive_group` 状态|默认施加失败|
|想替换现有同组状态|必须由技能/效果显式声明 `replace_existing = true`|
|同一状态重复施加|按该状态的 `stacking` 规则处理，默认不叠加|

## 3. 内置状态语义

### 3.1 主要状态

|状态|默认分组|默认持续方式|核心效果|
|---|---|---|---|
|中毒|`major_status`|`permanent`|回合末受到 `floor(max_hp / 8)` 伤害|
|灼伤|`major_status`|`permanent`|回合末受到 `floor(max_hp / 16)` 伤害；物理伤害进入 `final_damage_mod = 0.5`|
|麻痹|`major_status`|`permanent`|速度乘 `0.5`；技能/奥义在执行前有 25% 概率失败|
|睡眠|`major_status`|由技能效果定义|技能/奥义在执行前失败；手动换人仍可正常执行|

说明：

- 睡眠不再写死“固定 2 回合”；持续时间由施加它的技能/效果实例定义。
- 若未来需要“剧毒”“异常灼伤”等变体，按状态变体或 payload 规则扩展，不额外发明新大类。

### 3.2 易失状态

|状态|默认持续方式|核心效果|
|---|---|---|
|畏缩|`triggers = 1`，且仅限当前回合|仅能施加给 `action_window_passed == false` 的目标；使其本回合下一次技能/奥义失败并移除；不能传给本回合后来补位的新单位|

## 4. 状态对不同指令的影响

|状态|技能/奥义|手动换人|
|---|---|---|
|中毒|不阻断|不阻断|
|灼伤|不阻断|不阻断|
|麻痹|可能阻断|不阻断|
|睡眠|阻断|不阻断|
|畏缩|阻断|不阻断|

## 5. 离场原因

|`leave_reason`|定义|
|---|---|
|`manual_switch`|玩家主动换人|
|`forced_replace`|被技能/效果强制换下，但单位未倒下|
|`faint`|HP 归 0 导致离场|

## 6. 离场重置表

|战斗态|`manual_switch`|`forced_replace`|`faint`|说明|
|---|---|---|---|---|
|当前 HP / MP|保留|保留|HP=0；离场后不再参与结算|存活单位的 HP/MP 带下场|
|`major_status` 组状态|保留|保留|移除|当前参考官方：长期状态不会因普通换人自动清掉|
|易失状态（如畏缩）|移除|移除|移除|临时状态离场即清|
|能力阶段|移除|移除|移除|对齐官方主系列常规口径|
|个人护盾|移除|移除|移除|team/field 级护盾不在此列|
|锁定目标/蓄力标记/本行动临时标记|移除|移除|移除|不得跨离场保留|
|当前回合队列项|若尚未轮到则取消|若尚未轮到则取消|取消|见模块 02 的失败分类|
|team / field 效果|保留|保留|保留|由各自作用域与持续时间决定|
|已公开信息|保留为公开|保留为公开|保留为公开|如已揭示的被动持有物|

## 7. 换人与倒下触发顺序

### 7.1 主动换人

|步骤|规则|
|---|---|
|1|手动换人行动进入执行|
|2|旧单位触发 `on_switch`，并带 `leave_reason = manual_switch`|
|3|旧单位触发 `on_exit`|
|4|按离场重置表清理旧单位的离场型战斗态|
|5|新单位入场|
|6|新单位触发 `on_enter`|
|7|由入场触发的被动 / 道具 / field 效果进入同触发点结算|

### 7.2 强制换下（未倒下）

|步骤|规则|
|---|---|
|1|强制换下效果生效|
|2|旧单位触发 `on_switch`，并带 `leave_reason = forced_replace`|
|3|旧单位触发 `on_exit`|
|4|按离场重置表清理旧单位战斗态|
|5|替补立刻入场并触发 `on_enter`|

### 7.3 倒下离场

|步骤|规则|
|---|---|
|1|当单位 HP 归 0，立刻标记为倒下|
|2|倒下单位触发 `on_faint`|
|3|若本次倒下存在可归属来源，则对来源触发 `on_kill`|
|4|倒下单位触发 `on_exit`，并带 `leave_reason = faint`|
|5|按离场重置表清理战斗态|
|6|若该方仍有后备单位，则立刻执行强制补位|
|7|新单位入场并触发 `on_enter` 及入场衍生效果|

补充规则：

- 倒下不是主动换人，不会触发 `on_switch`。
- 强制补位不是普通行动，不进入行动排序。

## 8. 同窗双倒下的处理顺序

若同一结算窗口内双方同时倒下，固定按以下顺序处理：

1. 先把本窗内所有倒下单位都标记为 `fainted`。
2. 再处理这些单位的 `on_faint` / `on_kill` / `on_exit`。
3. 完成本窗全部倒下与离场清理后，再统一检查双方是否还有可上场单位。
4. 若双方都有后备，则双方都立刻补位。
5. 若双方都没有后备，则本窗直接判平局。

## 9. 回合节点中的状态结算

|节点|规则|
|---|---|
|`turn_start`|只处理当前在场单位|
|`turn_end`|只处理当前在场单位|
|离场单位|不参与 `turn_start / turn_end`|
|重新上场后|状态继续按剩余持续时间生效|

## 10. DOT、反伤与击杀归属

|场景|规则|
|---|---|
|中毒/灼伤在回合末致死|算状态来源方造成的击杀；若无来源记录，则记为系统伤害，不触发 `on_kill`|
|默认动作反伤致死|视为自伤；不把击杀归给对手|
|field 伤害致死|归给该 field 实例的创建者；若实例无创建者，按系统伤害处理|

## 11. 状态与换人的官方参考原则

当前在以下方面尽量贴近官方主系列：

1. 长期状态默认不会因为普通换人自动清除。
2. 畏缩只影响本回合还没轮到行动的单位。
3. 能力阶段和多数临时战斗态在离场后清空。

但以下是本项目主动保持的数据化设计：

1. 睡眠时长不写死全局值，由技能效果定义。
2. 状态互斥靠 `exclusive_group` 定义，不靠硬编码大类。
